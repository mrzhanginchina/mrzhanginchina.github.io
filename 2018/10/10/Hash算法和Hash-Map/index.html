Hash算法和 Hash-Map

Posted on 2018-10-10

Hash算法与Hash-Map:

什么是hash算法:

  • hash算法是将一系列的较长的二进制数据转换为较短的二进制数据的一种方法.
  • hash算法的两个特点: 不可逆和无冲突.
  • 不可逆: 当知道一个数据X的hash值之后, 不能根据hash数值反向推测出X的数值.
  • 无冲突: 如果两个数据X和Y的值不同, 则hash(X)和hash(Y)一定不相同.
  • 上面的两个特点在数学上都是不成立的, 目前通用的hash加密方法例如MD5和SHA等方法都是将逆向破解的代价变成无穷大.
  • hash算法的应用主要是校验和加密. 例如: 可以利用hash算法判断几个文件是否是同一个文件. 网站注册用户名和密码时候, 可以只保存用户密码的hash值, 通过这种方式可以有效地避免账号被盗用等不安全因素.

Hash-Map数据结构:

  • hash-map算法中, 空间一定是大于原始数据的空间大小的. 这点和hash算法正好相反, 因为hash算法是将大的数据空间映射到小的空间范围当中去.
  • 通过hash算法直接定位key值在数据库中的位置的数据结构叫做hash-map, hash-map的遍历代价可以看做是O(1), 是一种典型的空间换时间的数据结构.
  • 负载因子: 如果需要存储70个数据, 但是在内存中申请了100个空间, 那么我们可以说空间的负载因子是0.7.
  • hash-map中经常出现冲突现象, 即几个不同的key值一直映射到了相同的hash值上, 解决hash冲突主要有两种方法, 一种是hash链表, 即将固定空间变为一个链表, 如果hash值冲突, 在将新的数据附加在链表的尾部. 另外一种是hash函数双散列方法: 即在进行hash运算时候考虑碰撞次数, 通过这种方式一直找到不碰撞的存储空间.

经常用到的hash函数方法:

  • 直接寻址法：取key或key的某个线性函数值为散列地址。即H(key)=key或H(key) = a•key + b，当中a和b为常数(这样的散列函数叫做自身函数).
  • 数字分析法：分析一组数据，因此数字分析法就是找出数字的规律，尽可能利用这些数据来构造冲突几率较低的散列地址. 例如如果按照员工的的国家地来构造hash, 那么冲突的可能性非常大, 但是如果使用员工的出生的乡镇名字来构造hash, 那么冲突的可能性就大大的减小了.
  • 平方取中法：取key平方后的中间几位作为散列地址。
  • 除留余数法：取key被某个不大于散列表表长m的数p除后所得的余数为散列地址。即 H(key) = key MOD p, p<=m。不仅能够对keyword直接取模，也可在折叠、平方取中等运算之后取模。对p的选择非常重要，一般取素数或m，若p选的不好，非常容易产生同义词.
